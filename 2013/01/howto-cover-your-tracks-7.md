&#12288;&#12288;4天前发布了《用虚拟机隐匿公网IP》，好些个读者到博客留言，抱怨说讲得太简单。缺少傻瓜化的、带截图的配置教程。俺恭敬不如从命，今天再发一篇《用虚拟机隐匿公网IP（配置图解）》，竭尽所能，写得尽量傻瓜化。<br />&#12288;&#12288;至于前面那一篇，标题修改为《用虚拟机隐匿公网IP（原理介绍）》。<a name='more'></a><!--program-think--><br />&#12288;&#12288;顺便说一下：<br />&#12288;&#12288;前面那篇发到 G+ 上，貌似被顶到 G+ 热门榜，转发数很多。看来有不少网友关注“隐匿身份”这个话题 :) 所以俺今后会多花点时间，普及这方面的相关常识。<br /><br /><h2>★准备工作</h2><br /><h3>◇思想上的准备</h3><br />&#12288;&#12288;首先，如果你对虚拟机软件不太熟悉，<b><u>一定要先看完</u></b>俺之前写的《<a href="../../2012/10/system-vm-0.md">扫盲操作系统虚拟机</a>》。<br />&#12288;&#12288;其次，再把本系列的前一篇博文（链接在“<a href="../../2013/01/howto-cover-your-tracks-6.md">这里</a>”）<b>认真看完</b>。那篇博文是介绍“虚拟机隐匿公网IP”的原理，配有精美示意图 :)<br /><br /><h3>◇虚拟机软件的准备</h3><br />&#12288;&#12288;既然要用虚拟机隐匿公网IP，当然要先把虚拟机软件准备好。假如你不晓得该选哪种虚拟机软件，请看俺之前的一篇博文（请翻墙看“<a href="../../2012/11/system-vm-3.md">这里</a>”），专门介绍虚拟机软件的选择。<br />&#12288;&#12288;本文主要拿 VMware Workstation（以下简称 VMware）和 VirtualBox 来介绍。<br /><br /><h3>◇代理软件的准备</h3><br />&#12288;&#12288;先声明一下，本文提到的“代理”一词是广义的，包括：普通代理、VPN、多重代理。<br />&#12288;&#12288;由于本教程是拿虚拟机跟代理软件进行组合搭配，所以你还得懂得使用代理软件。这点应该不难——只要你有翻墙的经历，你就已经在同"代理软件"打交道了。如果你从来没玩过翻墙，请先学习俺博客上的诸多翻墙教程（包括：TOR、I2P、赛风、世界通、自由门、无界 ......）。<br />&#12288;&#12288;关于代理的类型，俺强烈建议用<b>多重代理</b>（教程在“<a href="../../2012/03/howto-cover-your-tracks-5.md">这里</a>”，需翻墙）。为啥捏？如果你对隐匿性的要求不高，根本都不需要看本教程。你来看本教程，就说明你对隐匿性有较高的要求。既然如此，当然要用多重代理啦——这可以大大增加逆向追踪的难度。<br />&#12288;&#12288;注意事项：<span style="color:red;"><b>要确保你用的代理软件，是监听在 0.0.0.0 地址，而不是监听在 127.0.0.1 地址</b></span>。如果代理软件只监听在 127.0.0.1 地址，那么其它虚拟机的网络软件是无法连接到这个监听端口的。<br />&#12288;&#12288;如何看代理软件在哪个地址上监听捏？可以在 Windows 的命令行窗口（就是那黑窗口）中执行如下命令，就可以看到当前系统中开启的所有监听端口以及该监听绑定的地址。<br /><b>netstat -an | find "LISTEN"</b><br />&#12288;&#12288;那么，万一你的翻墙工具的监听端口没有绑定到 0.0.0.0 该咋办捏？别担心，俺后面又专门写了一篇教程《<a href="../../2013/01/cross-host-use-gfw-tool.md">多台电脑如何共享翻墙通道</a>》，教你如何解决监听端口绑定地址的问题。<br /><br />&#12288;&#12288;如果你光使用 VPN 作为“代理”。由于 VPN 本身是不开启监听端口的。那么你就必须想办法共享 VPN 的翻墙通道。至于如何共享，请看《<a href="../../2013/01/cross-host-use-gfw-tool.md">多台电脑如何共享翻墙通道</a>》。<br /><br /><h3>◇关于操作系统的说明</h3><br />&#12288;&#12288;本教程适用于目前的各种主流操作系统，包括但不限于 Windows、Linux、Mac OS X ......<br />&#12288;&#12288;考虑到目前 Windows 系统的用户占绝大多数，本教程拿 Windows 系统来说事儿。希望 Linux 系统和 Mac OS X 系统的用户别怨俺偏心。<br />&#12288;&#12288;注意事项：<span style="color:red;"><b>要特别小心真实系统和虚拟系统的防火墙设置</b></span>。很多人是因为防火墙没设好，导致代理无法连通。<br /><br /><h2>★真实系统（Host OS）的物理网卡</h2><br />&#12288;&#12288;先跟大伙儿说一下：今天这个教程跟物理网卡【<b>没有</b>】半毛钱关系。跟你电脑上装了多少块物理网卡，也【<b>没有</b>】半毛钱关系。在本文后续的介绍中，<b>不会</b>再涉及到物理网卡。待会儿在配置代理的时候，也<b>不会</b>再涉及物理网卡上的 IP 地址。切记！<br />&#12288;&#12288;另外，今天这个教程，跟你用的上网方式也没有关系。不论你是在公司上网还是在家用宽带，本教程都适用。<br /><br /><h2>★真实系统（Host OS）的虚拟网卡</h2><br />&#12288;&#12288;【<b>这部分是重点，注意看喽！</b>】<br /><br /><h3>◇代理软件的安装</h3><br />&#12288;&#12288;在单虚拟机方案中，代理直接安装在 Host OS 里面。关于代理软件的安装，就不用再啰嗦了吧？<br /><br /><h3>◇上网软件的配置</h3><br />&#12288;&#12288;（<b>这部分也是重点，看仔细喽！</b>）<br />&#12288;&#12288;为网络软件配置代理的时候，通常要填写代理的 IP 地址和端口号。端口号通常不会搞错。因为每一款代理软件开启的端口号是固定的。但是 IP 地址常常会填错。很多人就是栽在这一步。<br />&#12288;&#12288;填写代理的 IP 地址，千万【<b>不能</b>】填 127.0.0.1，因为这个地址表示本机，也就是 Guest OS 自己。<br 地址头三位是一样的，<span style="color:red;"><b>第四位不能相同——否则 IP 地址会冲突</b></span>。<br /><b>子网掩码</b><br />填写 255.255.255.0<br /><b>默认网关</b><br />不用填<br /><b>DNS</b><br />不用填<br /><br /><h3>◇代理软件的安装</h3><br />&#12288;&#12288;在双虚拟机方案中，<b>代理是安装在“虚拟系统B”（网关）里面的</b>，别搞混喽！关于代理软件的安装，就不用再啰嗦了吧？<br /><br /><h3>◇上网软件的配置（以浏览器为例）</h3><br />&#12288;&#12288;在双虚拟机方案中，那些有危险的<b>上网软件是安装在“虚拟系统A”里面的</b>，别搞错喽！<br />&#12288;&#12288;下面详细说说上网软件的代理配置。<br />&#12288;&#12288;为网络软件配置代理的时候，通常要填写代理的 IP 地址和端口号。端口号通常不会搞错。因为每一款代理软件开启的端口号是固定的。但是 IP 地址常常会填错。所以<b>请大伙儿把下面看仔细喽！</b><br />&#12288;&#12288;填写代理的 IP 地址，<b>千万不能</b>填 127.0.0.1，因为这个地址表示本机，也就是“虚拟系统A”自己。<br />&#12288;&#12288;<b>正确的写法</b>是：填写另一个虚拟系统（虚拟系统B）的那个 <b>Host-Only</b> 网卡的 IP。<br />&#12288;&#12288;因为“虚拟系统B”（网关）有两块网卡，很多人填错了，栽倒在这一步。俺以 IE 浏览器为例，截图如下<br /><img src="../../images/2013/01/E4_zkEEFoQwmHuHdp9JOps50_4GNu2yfL9mxUm8uQUE3nsbzVd6p3xNSCdGlQrgLhP1VMjVeg8hsPLQ8x8wEt-oIyJDBpA5KVZM3-6tVPuFsn6dyCTENbWAq0Y8" alt="不见图 请翻墙"><br /><br /><h3>◇注意事项</h3><br />&#12288;&#12288;在双虚拟机方案中，虚拟机在启动时有可能会出现错误提示：“网络上有重名”。如果你碰到这个错误，只需把虚拟机网卡的 NetBIOS 功能禁用，即可解决。如何禁用 NetBIOS，请看“<a href="http://technet.microsoft.com/zh-cn/library/ms143696%28v=sql.90%29.aspx" target="_blank" rel="nofollow">微软官网这里</a>”。<br /><br /><h2>★双虚拟机方案（NAT+Internal）</h2><br />&#12288;&#12288;（经热心读者 meek 提醒，俺补充了这一章节）<br />&#12288;&#12288;在《扫盲操作系统虚拟机》系列的<a href="../../2012/12/system-vm-5.md">第5篇</a>，俺介绍了 Guest OS 的各种网卡模式，其中也包括“Internal 模式”。这种模式类似于“Host Only 模式”，差别在于——Host OS 上的进程无法看到“Internal 模式”的虚拟网卡。<br />&#12288;&#12288;因此，“双虚拟机方案”也可以采用“NAT+Internal”的玩法。这种玩法的好处是——隔离性更好；缺点是——VMware 不支持这种模式（但 Virtual Box 支持）。<br />&#12288;&#12288;如果你想采用“NAT + Internal”的方式配置虚拟网卡，那么整个配置过程跟前一章节介绍的“NAT + HostOnly”很相似，差别仅仅在于把两个虚拟网卡的“HostOnly 模式”替换为“Internal 模式”。所以俺就不重复罗嗦了。<br /><br /><h2>★验证虚拟机的隔离性</h2><br />&#12288;&#12288;无论是“双虚拟机方案”还是“单虚拟机方案”，配置完毕，联通代理之后，为了保险起见，你需要再验证一下虚拟机的隔离性。<br />1. 先把代理关闭，到虚拟机A 中运行危险软件，看它是否能联网成功。（如果联网成功，说明你的配置有误）<br />2. 进入虚拟机A，开启一个命令行窗口，用 ping 命令测试一下你常用的 DNS 服务器的 IP（如果能 ping 到，说明你的配置有误）<br /><br /><h2>★结尾</h2><br
